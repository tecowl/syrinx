SVM - Small Version Manager ver 0.1.2

SVMはディレクトリ監視型バージョン管理ツールです。

[基本的な使い方]
BaseDir(基準ディレクトリ): 相対パスの元になるディレクトリ
WorkDir(作業ディレクトリ): バージョンファイルを保持するディレクトリ
MonitorDir(監視ディレクトリ): ファイルの変更を監視するディレクトリ

以上３つを設定しメニューの[実行|監視開始]を行うだけで、監視ディレクトリ内の
ファイルを監視し、変更があった場合に作業ディレクトリにそのコピーを作成します。

＜注意＞
基準ディレクトリは必ず、監視ディレクトリと作業ディレクトリをサブディレクトリと
する両者の上位のディレクトリを指定してください。


[細かい設定]
SaftyInterval(安全間隔): SVMの行うディレクトリの監視は、OSからのファイルの変更
通知を受け取り、バックアップをコピーする処理などが行われますが、OSは一つの
ファイルの変更でも複数の通知を行う場合があります。それらを無視するために、
最初に通知を受け取ってからｎ秒間は通知を無視する、という設定をSaftyIntervalで
行います。単位は[秒]です。

AttrMask(属性マスク): 新しいファイルや変更されたファイルを検索する際に使用され
るマスクです。各項目はor条件で結ばれます。ただし、k0faDirectoryはチェックしても
無視されます。

k0faReadOnly	読み出し専用ファイル
k0faHidden	非表示ファイル
k0faSysFile	システムファイル
k0faVolumeID	ボリュームファイル
k0faDirectory	ディレクトリファイル
k0faArchive	アーカイブファイル

Masks(ファイル名マスク): 新しいファイルや変更されたファイルを検索する際に使用され
るマスクです。各項目はor条件で結ばれます。
MasksとAttrMaskはand条件で結ばれます。


SubDir(サブディレクトリ検索): 新しいファイルや変更されたファイルを検索する際に
サブディレクトリを検索するかどうかを設定するチェックです。チェックしてあれば、
サブディレクトリを検索します。



[Original Files]ページは、監視対象となっているファイル名の一覧を表示します。
ファイル名をダブルクリックすると、履歴画面が表示されます。

[Save Points]ページには、監視対象となっているファイルがいつバックアップされた
のかを示すグリッドが表示されます。

[Original Files]ページと[Save Points]ページはメニューの[表示|最新の情報に更新]
によって表示が更新されます。BaseDirとWorkDirが設定されていれば、監視を行わなく
ても履歴を見ることが出来ます。


[History]フォームでは、２つのバージョンを選択して[Show Difference]を行うと、
右側のペインに差異が色付きで表示されます。
１つのバージョンを選択して[Show Difference]を行うと、現在のファイルとの差異が
色付きで表示されます。
[Trimed Compare]チェックボックスがチェックされていると、差異を検索する際に文字
列のトリムを行ってから、文字列の比較を行います。


[既知バグ]
・Masksの入力時にピリオドを入力すると先頭にカーソルが移動してしまう。
・SavePointsページでディレクトリ名を表示している結合したセルがスクロール領域の
中に入りきっていない場合に正しく表示されない。セルが全て見えている場合なら問題
ない。


[追加予定機能]
・VSSのラベルのようなバージョンファイルのインデックス機能
・不要なバージョンファイルやセーブポイントを削除
