c0UnifiedObjCtrl

クラス図



コンポーネントパレットにあるTc0UnifiedObjCtrlコンポーネントは、Tc0ObjCtrlBridgeにTc0CollectionGridを使用できるように機能を追加したものです。

目的

任意のオブジェクトのプロパティをフォーム上のコントロールなどにマッピングするためのコンポーネントです。
プロパティが変更された場合にコントロールの表示をその値に変更したり、コントロールに対する操作によってプロパティの値を変更することができます。
ただし、マッピングされるプロパティはpublished宣言されている必要があります。 

簡単な使い方

まず、簡単な例としてコンポーネントのプロパティをフォームに表示してみましょう。

１．              新規作成したフォームにTc0UnifiedObjCtrl,TScrollBox,TEditを貼り付けてください。

２.c0UnifiedObjCtrl1のBuilderプロパティのCtrlParentプロパティをScrollBox1を設定します。 c0UnifiedObjCtrl1のTargetComponentプロパティをEdit1に設定します。

４．フォーム上のc0UnifiedObjCtrlを右クリックし、[コントロールの生成]を選択します。

[コントロールの生成]が表示されない場合は、コンポーネントエディタが正しくインストールされていない可能性があります。「インストールガイド」を参照し、インストールしてください。

5.ScrollBoxの上に、コンポーネントが生成されます。

実行すると各種プロパティを変更することが確かめることができます。

例えば、VisibleのチェックをFalseに設定すると、Edit1は非表示になり、AlignmentのコンボボックスでtaRightJustifyを選択するとEdit1の文字列は右寄せになります。

 

 ちなみに、設計時にプロパティを何も設定せず、コードで同じ設定を行うと以下のようになります。

 

procedure TForm1.FormCreate(Sender: TObject);
begin
  c0UnifiedObjCtrl1.Builder.CtrlParent := ScrollBox1;
  c0UnifiedObjCtrl1.TargetComponent := c0Edit1;
  c0UnifiedObjCtrl1.BuildCtrls;
end;

 

応用的な使い方

[簡単な使い方]ではすべてのプロパティを表示/変更できるようになっていましたが、あるプロパティについては対象外としたい場合もあります。ここでは、そのような場合の操作方法を紹介しますが、その前にTc0UnifiedObjCtrlがどのように動作するのか概要を説明します。

Tc0UnifiedObjCtrlの動作の概要

TargetComponentプロパティ(Targetプロパティ)が設定されると、c0UnifiedObjCtrlのProtoTypesプロパティにはターゲットとして指定されたインスタンスのpublishedのプロパティを表示するための項目(以後”ProtoType”)が追加されます（ただし、ProtoTypesに項目が一つも追加されていない場合だけです。一つでも項目があるならば、TargetComponentプロパティを設定しても項目は追加されません）。

この時点では、コントロールは生成されていません。

ProtoTypesプロパティの[･･･]を開くと・・・

ProtoTypeがプロパティの分だけ追加されていることが分かります。

 

コントロールは、このProtoTypeの設定に従って生成されます。対応するProtoTypesがなければコントロールは設定されません。

 もし、TargetComponentプロパティ(Targetプロパティ)を設定する前に、対象外とするプロパティが分かっているのならば、Tc0UnifiedObjCtrlのDisabledPropsにそのプロパティ名を追加してください。

 [簡単な使い方]で、Nameプロパティと、TagプロパティのProtoTypeが追加されていないのは、DisabledPropsにデフォルトで”Name”、”Tag”が追加されているからです。もし、これらのコントロールの表示を行いたいという場合には、DisabledPropsから削除してください。
ただし、”Name”についてはお勧めできません。

各ProtoTypeには、どのようなコントロールを生成するのかを決めるプロパティがあります。

　

ProtoTypeの主なプロパティ

Ctrl

生成されたコントロールを示します。（上図では、まだ[コントロールの生成]が行われていないため、nilのままになっています）
自動生成する前にコントロールを指定したい、あるいは生成後にコントロールを変更したいという場合には、このプロパティによって変更してください。

CtrlClassName

コントロールを生成する場合に、どのクラスのコントロールを生成するのかを指定します。このプロパティは文字列ですので、生成する前に他のクラスにしたい場合には、このプロパティを変更してください。

CtrlParent

コントロールを生成する時に、どのコントロールを親として生成するのかを指定します。
通常はBuilderのCtrlParentで指定しますが、項目ごとに異なる場合には、こちらのプロパティを使用してください。

AutoUpdateKinds

コントロールのどのようなイベントで、値をTargetに反映させるかを指定できます。

WithoutCtrl

コントロールを生成しないかどうかを指定します。デフォルトはFalseで、Falseならばコントロールを生成し、Trueならば、生成しません。

 表示したくない項目 

１、Targetプロパティ(TargetComponentプロパティ)を設定する前

この時点で、表示したくないプロパティの名前がわかっているのならば、Tc0UnifiedObjCtrlのDisabledPropsにそのプロパティ名を追加してください。
これにより、Targetプロパティに値が設定されても、指定した名前のプロパティからProtoTypeが作られることはありません。

２、  TargetComponentプロパティを設定し、[コントロールの生成]を行う前

この場合には、ProtoTypeのWithoutCtrlをTrueに設定することでコントロールの生成を防ぐことができます。もし、ProtoType自身が不必要ならば、コレクションエディタ上でそのProtoTypeを削除しても構いません。

３、設計時に[コントロールの生成]を行った後

この場合、表示したくないコンポーネントをフォームから削除してください。通常のコントロールと同様、DELキーで削除できます。２と同様、ProtoType自身が不必要ならば、コレクションエディタ上でそのProtoTypeを削除しても構いません。  

オブジェクト指向的使用法

 応用的な使い方として、クラスを作成し、Tc0UnifiedObjCtrlを使用して作成したクラスのインスタンスをフォームに表示してみましょう。

 １．クラスを宣言します。

新しいアプリケーションのフォームのinterface部に以下のように入力してください。

   TSampleObj = class(TPersistent)
  published
    property Value: Integer;
    property Text: string;
  end;

ここで、[Ctrl+Shift+C]を押すと、自動的にメソッド、変数などが生成され、implementation部にはメソッドの実装が追加されます。

３．              フォームのprivateに

FObj: TSampleObj;

という宣言を追加します。

４．              c0UnifiedObjCtrlの動作を確認するために、以下のようにフォームにコンポーネントを貼り付けてください。

それぞれのボタンのOnClickイベントのイベントハンドらとなるメソッドには、以下のコードを記述してください。

インスタンスの生成

 
procedure TForm2.c0Button1Click(Sender: TObject);
begin
  if FObj = nil then
    FObj := TSampleObj.Create(nil);
end;

インスタンスの破棄

 
procedure TForm2.c0Button2Click(Sender: TObject);
begin
  FreeAndNil(FObj);
end;

c0UnifiedObjCtrl1への設定

 
procedure TForm2.c0Button3Click(Sender: TObject);
begin
  c0UnifiedObjCtrl1.Builder.CtrlParent := ScrollBox1;
  c0UnifiedObjCtrl1.Target := FObj;
  c0UnifiedObjCtrl1.BuildCtrls;
end;

c0UnifiedObjCtrl1への設定解除

 
procedure TForm2.c0Button4Click(Sender: TObject);
begin
  c0UnifiedObjCtrl1.ClearCtrls;
  c0UnifiedObjCtrl1.ProtoTypes.Clear;
  c0UnifiedObjCtrl1.Target := nil;
end;

インスタンスの内容を表示

 
procedure TForm2.c0Button5Click(Sender: TObject);
var
  ms1, ms2: TMemoryStream;
begin
  ms1 := TMemoryStream.Create;
  try
    ms1.WriteComponent(FObj);
    ms2 := TMemoryStream.Create;
    try
      ms1.Position := 0;
      ObjectBinaryToText(ms1, ms2);
      ms2.Position := 0;
      Memo1.Lines.LoadFromStream(ms2);
    finally
      FreeAndNil(ms2);
    end;            
  finally
    FreeAndNil(ms1);
  end;
end;

 

４．実行して以下のように操作してみてください。

[インスタンスの生成]-[c0UnifiedObjCtrl1への設定]-[Text の エディットになんらかの文字列を入力する]-[インスタンスの内容を表示]
すると、Memo1にはTSampleObj型のFObjで参照されるインスタンスのデータが表示されます。エディットに入力した値がMemo1に表示されるはずです。

[インスタンスの破棄]を行った後、エディットに値を入力しようとすると、EAccessViolation例外が発生します。実際のアプリケーションでは、このような現象は致命的と言えるので、[インスタンスの破棄]を行ったら、[c0UnifiedObjCtrl1への設定解除]で行っているように、Targetプロパティにnilを設定してください。

ちゃんとTSampleObj.SetTextメソッドが呼び出されていることも確認してください。SetTextメソッドにブレークを設定することで確認できます。

詳細  

メソッド

BuildCtrls、ClearCtrls

public procedure BuildCtrls;
public procedure ClearCtrls;
BuildCtrlsメソッドは、ProtoTypesの各ProtoTypeに従って、コントロールを生成します。
 ClearCtrlsメソッドは、ProtoTypesの各ProtoTypeのCtrlプロパティに設定されているコントロールをすべて破棄し、nilに設定します。

LoadFromTarget、SaveToTarget

public procedure LoadFromTarget;
public procedure SaveToTarget;
LoadFromTargetメソッドは、Targetプロパティに設定されたインスタンスの各プロパティの値を対応するコントロールに設定します。
SaveToTargetメソッドは、Targetプロパティに設定されたインスタンスの各プロパティの値を、対応するコントロールから読み取って、設定します。

AutoUpdateプロパティがFalseに設定されている場合には、これらのメソッドを用いて、コントロールからTarget、Targetからコントロールへの値の反映を明示的に行ってください。

プロパティ

TargetとTargetComponent

 
public property Target: TObject;
published property TargetComponent: TComponent

Targetプロパティは、表示対象となるインスタンスを表します。このプロパティにコントロールにマッピングしたいインスタンスを設定することでマッピングが可能です。
TargetComponentプロパティは、設定時にTargetプロパティにアクセスするためのもので、型がTComponent型であることと、可視性がpublishedであることを除けばTargetプロパティと同じものであると言えます。
コードにおいて、どちらを使用するのかを迷った場合には、Targetプロパティを使用してください。
Targetプロパティ(TargetComponentプロパティ)に有効なオブジェクトへの参照を設定すると、Prototypesプロパティ(後述)に項目が１つも設定されていない場合、自動的にPrototypesプロパティに項目が追加されます。  

AutoUpdate

published property AutoUpdate: Boolean
コントロールに、ProtoTypeのAutoUpdateKindsプロパティで示されたイベントが発生した場合に、自動的にTargetのプロパティに値を反映するかどうかを示すプロパティです。

DisabledProps

published property DisabledProps: TStrings
Tc0UnifiedObjCtrlは、TargetプロパティあるいはTargetComponentプロパティに有効な参照が設定されたときに、ProtoTypesに項目が一つもなければ、自動的にPotoTypeを生成し、ProtoTypesに追加しますが、その際、項目を追加しないプロパティ名をDisabledPropsプロパティに設定します。

このプロパティに、無効なプロパティ名が追加されていても問題ありません。デフォルトで、”Name”と”Tag”が設定されています。 

ProtoSourceCompo、ProtoSourcePropName

published property ProtoSourceCompo: TComponent
published property ProtoSourcePropName: string
Targetととして、あるコンポーネントのプロパティとして表されるインスタンスを設計時に指定したい場合に使用します。
ProtoSourceCompoは、インスタンスをプロパティとして保持するコンポーネント、ProtoSourcePropNameはそのプロパティ名です。  

Builder

published property Builder: T0ObjCtrlBuilder
コントロールの生成を行うためのオブジェクトです。プロパティを変更することで生成を行う際の設定を変更することができます。  

主なプロパティ

published property CtrlMargin: Integer
生成するコントロールのY座標の間隔を指定できます。デフォルトは4です。


published property CtrlOwner: TComponent
published property CtrlParent: TWinControl
生成するコントロールのオーナーコンポーネント(CtrlOwner)と、親コントロール(CtrlParent)を指定することができます。CtrlOwnerはフォーム上の場合デフォルトでそのフォームになります。CtrlParentはデフォルトで設定されませんので、手動で設定してください。

published property CtrlOfTypeClassName: string
published property CtrlOfType: T0CtrlOfType
どのコントロールを生成するのかを決定するオブジェクトを表します。CtrlOfTypeClassNameでCtrlOfTypeのクラスを変更することができますが、通常あまり変更しないことをお勧めします。

ProtoTypesプロパティ

published property ProtoTypes: T0ProtoTypeCtrls
コントロールを生成するためのProtoTypeを複数保持します。
ProtoTypeの主なプロパティについては、[応用的な使い方| Tc0UnifiedObjCtrlの動作の概要| ProtoTypeの主なプロパティ]を参照してください。
